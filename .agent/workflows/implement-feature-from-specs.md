# Implement Feature from Specs

## Purpose

Build one feature from `specs.md` in a safe, structured way.

## Prompt

1. Read `README.md`, `RULES.md`, `context.md`, `memory.md`, and the relevant feature section in `specs.md`.
2. Inspect the current implementation before proposing changes.
3. Extract:
   - goal
   - functional requirements
   - non-functional requirements
   - constraints
   - acceptance criteria
4. Produce an Implementation Plan Artifact with:
   - user outcome
   - data/state model
   - exact files to create or modify
   - edge cases
   - testing plan
5. Wait for approval.
6. After approval, implement the feature in small steps.
7. Add or update tests.
8. Update `memory.md`.
