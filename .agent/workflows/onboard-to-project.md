# Onboard to Project

## Purpose

Understand the current repository quickly and propose the highest-value next step.

## Prompt

Read:
- `README.md`
- `AGENTS.md`
- `RULES.md`
- `context.md`
- `memory.md`
- `specs.md`

Then inspect the repository and:

1. Summarize the current architecture.
2. Identify entry points, routes, state, styling, and storage setup.
3. Compare the repo to the intended Boredom Gym structure.
4. List the biggest missing pieces.
5. Produce an Implementation Plan Artifact for the best next step.
6. Wait for approval before broad multi-file changes.
