# Safe Refactor

## Purpose

Improve maintainability without changing intended behavior.

## Prompt

1. Read `RULES.md`, `context.md`, and `memory.md`.
2. Inspect the relevant files.
3. Explain current structure and why it is becoming hard to maintain.
4. Create an Implementation Plan Artifact with:
   - refactor goal
   - files and symbols to touch
   - behavior that must remain unchanged
   - risk assessment
   - testing updates needed
5. Wait for approval.
6. After approval, refactor in small logical steps.
7. Update `memory.md`.
